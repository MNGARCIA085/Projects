1. Introduction ¶

In this notebook, we implement and train a Siamese neural network using the triplet loss function, a powerful combination commonly used in tasks where measuring the similarity between inputs is more important than classifying them into fixed categories.

Objective

Our goal is to train a model that learns to project inputs (e.g., images, texts, embeddings) into a feature space where similar samples are closer together and dissimilar samples are farther apart. This is particularly useful for:

  • Face recognition and verification

  • Signature or fingerprint matching

  • Recommender systems

  • Metric learning applications

Why Triplet Loss?

Unlike contrastive loss (which uses pairs), triplet loss works by comparing anchor-positive-negative triplets:

  • Anchor: A reference sample

  • Positive: A sample of the same class as the anchor

  • Negative: A sample of a different class

The loss encourages the distance between anchor and positive to be smaller than the distance between anchor and negative by at least a specified margin.

This encourages the network to learn discriminative embeddings that respect the desired similarity relationships.

# Improved triplet creation
def create_triplets(x, y, num_triplets=5000):
    triplets = []
    num_classes = 10
    class_indices = [np.where(y == i)[0] for i in range(num_classes)]
    
    for _ in range(num_triplets):
        anchor_class = np.random.randint(0, num_classes)
        while len(class_indices[anchor_class]) < 2:
            anchor_class = np.random.randint(0, num_classes)
        
        anchor_idx, pos_idx = np.random.choice(class_indices[anchor_class], 2, replace=False)
        
        neg_class = np.random.randint(0, num_classes)
        while neg_class == anchor_class or len(class_indices[neg_class]) == 0:
            neg_class = np.random.randint(0, num_classes)
        neg_idx = np.random.choice(class_indices[neg_class])
        
        triplets.append([x[anchor_idx], x[pos_idx], x[neg_idx]])
    
    return np.array(triplets)

# Create triplets

triplets_test = create_triplets(x_test, y_test, num_triplets=2000)

4. Build and compile the model ¶

First, lets create the embedding network.

The embedding network is the core of the Siamese architecture. Its a neural network that maps raw input images into a dense vector space (i.e., embeddings) where semantic similarity corresponds to spatial closeness.

In this case, it will map each MNIST image (e.g., a 28x28 digit) to a 64-dimensional normalized vector.

The idea is that:

  • Similar digits (e.g., two 3s) should have embeddings that are close together.

  • Dissimilar digits (e.g., a 3 and an 8) should have embeddings far apart.

In [10]:
# Base embedding network
def build_embedding_network(input_shape):
    inputs = layers.Input(shape=input_shape)
    x = layers.Conv2D(32, 3, activation='relu')(inputs)
    x = layers.MaxPooling2D()(x)
    x = layers.Conv2D(64, 3, activation='relu')(x)
    x = layers.MaxPooling2D()(x)
    x = layers.Flatten()(x)
    x = layers.Dense(64)(x)
    x = layers.Lambda(lambda x: tf.math.l2_normalize(x, axis=1))(x)
    return Model(inputs, x)

We will create the following model:

In [11]:
# Siamese model that outputs distances
def build_siamese_model(embedding_network):
    anchor_input = layers.Input(shape=input_shape)
    positive_input = layers.Input(shape=input_shape)
    negative_input = layers.Input(shape=input_shape)
    
    a_embed = embedding_network(anchor_input)
    p_embed = embedding_network(positive_input)
    n_embed = embedding_network(negative_input)
    
    pos_dist = tf.reduce_sum(tf.square(a_embed - p_embed), axis=1, keepdims=True)
    neg_dist = tf.reduce_sum(tf.square(a_embed - n_embed), axis=1, keepdims=True)
    
    # Stack distances along last axis
    distances = layers.Concatenate(axis=-1)([pos_dist, neg_dist])
    return Model([anchor_input, positive_input, negative_input], distances)

Triplet Loss

The triplet loss ensures that an anchor image is closer to a positive image (same class) than to a negative image (different class), by at least a margin:

$$ \mathcal{L} = \max\left( \|f(a) - f(p)\|_2^2 - \|f(a) - f(n)\|_2^2 + \alpha, \, 0 \right) $$

Where:

  • ( f(x) ): the embedding function (neural network)
  • ( a ): anchor image
  • ( p ): positive image (same class)
  • ( n ): negative image (different class)
  • ( \alpha ): margin (e.g., 0.5)
In [14]:
# Triplet loss function
def triplet_loss(y_true, y_pred):
    pos_dist = y_pred[..., 0]  # Take first distance
    neg_dist = y_pred[..., 1]  # Take second distance
    margin = 0.5
    loss = tf.maximum(pos_dist - neg_dist + margin, 0.0)
    return tf.reduce_mean(loss)

# Get the shape of a single input image (e.g., (28, 28, 1) for MNIST)

Why do we need dummy outputs in model.fit:

The reason is that Keras always expects both:

  • Inputs: your training data

  • Targets (labels): what the model is trying to predict

However, in a Siamese network trained with triplet loss, youre not predicting a label like a class index (e.g., "3" or "9"). Instead:

  • The model computes the embedding distances between anchor-positive and anchor-negative.

  • The loss function itself (triplet loss) compares these distances and computes how much to penalize the model there's no need for ground-truth labels.

But Keras still requires a y_true argument for consistency, even if the loss doesn't use it.

What to Expect?¶

  • Clusters Formed by Digit Classes
    Because the network is trained with triplet loss to pull embeddings of the same digit closer and push embeddings of different digits apart, we should see distinct clusters in the 2D plot, where each cluster corresponds roughly to one digit class (0 through 9).

  • Separation Between Clusters
    Ideally, the clusters for different digits will be well separated, reflecting that the network learned a meaningful embedding space where digits are distinguishable by their features.

  • Some Overlap or Noise
    Since MNIST digits can be similar (like '4' and '9' or '3' and '5'), you might see some overlap or fuzziness at the borders of clusters thats normal, especially with TSNE projections.

  • Compactness of Clusters
    The tighter the clusters, the better the embedding network learned to represent digit similarity. Loose or scattered clusters could mean the model needs more training or a different architecture.
